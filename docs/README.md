# Frontend Documentation

Developer notes and learnings from building the Expense Tracker frontend.

## 📁 Structure

```
docs/
├── README.md           # This file — index of all notes
├── images/             # Screenshots and diagrams
├── react-patterns.md   # React component and hook patterns
├── zustand-state.md    # State management notes (now uses Redux Toolkit)
├── tailwind-tips.md    # Tailwind CSS utility tricks
├── vite-config.md      # Vite setup, chunking, and PWA plugin config
└── api-handling.md     # Axios client, interceptors, and error handling
```

## 🖼️ Adding Screenshots

1. Save screenshots in the `images/` folder.
2. Reference them in docs with a relative path:
   ```markdown
   ![Description](./images/screenshot-name.png)
   ```

## ✍️ Conventions

- One file per topic.
- Use headings, code blocks, and inline screenshots to explain concepts.
- Keep notes concise — link to official docs for deep reference material.
